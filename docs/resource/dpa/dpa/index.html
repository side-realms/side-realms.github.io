🏝️ ここではサイドチャネル解析 (SCA) について，その概要を紹介します 🏝️

回路からの情報リーク#

デバイスの中には回路が実装されています． その回路は論理ゲートから構成されています． 論理ゲートは boolean 変数を扱うので，入力が 1, 0 かつ出力が 1, 0 となります． さらに，論理ゲートは CMOS から構成されており，この CMOS はトランジスタで構成されています．
この CMOS はビットフリップが起こると消費電力が増える特性をもちます． 通常動作のとき，つまりビットフリップが起こらないとき (0 -> 0, 1 -> 1) は消費電力が少なく， ビットフリップが起こるとき (0 -> 1, 1 -> 0) に消費電力が増えます． このとき，CMOS 全体の消費電力は，何もしなくても消費する電力 (static 消費電力) と，ビットフリップで消費する電力 (dynamic 消費電力) の和で表されます． $$ P_{CMOS} = P_{static} + P_{dynamic} $$

CMOS の説明#

static 消費電力#

何もしなくても，電源電圧を印可するだけで CMOS が消費する電力を static 消費電力といいます． ここでキャパシタを考えてみます． キャパシタは，ふたつの離れた導体 (conductor) が違う電位にある場合，両者の間にキャパシタンスが発生します． 導体の間には誘電体 (dielectric) があり，この誘電体には全く電流が流れないことが理想です． しかし，現実の誘電体は完全には絶縁体となっておらず，leakage current と呼ばれる電流がわずかに流れます． MOSFET は，ゲート・ドレイン間およびゲート・ソース間に寄生容量が発生しますが，その間にある絶縁体がわずかに電流を流すことで static 消費電力が発生します． このとき，static 消費電力は以下のように表されます． $$ P_{static} = I * V $$

dynamic 消費電力#

CMOS の動作を観察すると，出力ビットが遷移するとき，つまりビットフリップがおこるときに pMOS と nMOS の ON/OFF が切り替わることが分かります． CMOS は，pMOS と nMOS で電源と GND 間の直流電流経路をもたないことが知られていますが，pMOS と nMOS で ON/OFF が切り替わるときに一瞬だけ両方同時に通電するタイミングが存在します． このときに流れる電流を貫通電流と呼び，dynamic 消費電力の一つの原因となっています．
また，CMOS のビットフリップは一瞬で起こらず，ゲートの充電時間が余分にかかります． このときに流れる電流が dynamic 消費電力の原因のひとつとなっています． 一回の充電電荷 Q は以下のようにあらわされます． $$ Q = C_L V_{DD} $$ このときC_L*は pMOS もしくは nMOS のキャパシタンス，V_{DD} は電源電圧です． すると，消費電力は以下のようにあらわされます． $$ P = Q * V_{DD} $$ さらに，これはクロックにも依存しています．一秒あたりのビットフリップの回数が多いほど，多くの電力を消費します． ただし，全てのクロックで切り替わるとは限らないので，クロック周波数 f に係数 α を掛ける必要があります． $$ P_{dynamic} = αf C_L V_{DD}^2 $$

CMOS の消費電力#

CMOS の消費電力は，以下のように表されるのでした． $$ P_{CMOS} = P_{static} + P_{dynamic} $$ このとき，static 消費電力は dynamic 消費電力に比べて小さいため，CMOS の消費電力は dynamic 消費電力に大きく左右されることになります． さらに，この dynamic 消費電力は，データがビットフリップする数に依存するため，最終的に CMOS 全体の消費電力はデータに依存することになります．

消費電力の設計#

この CMOS ビットフリップの考え方を利用すると消費電力をシミュレーションすることができます． 例えば 8 bit のレジスタを考えてみます． 以下のようにビットフリップが起こったとします． $$(0000 0000) \rightarrow (0110 0010)$$ このとき 3 bit 分のビットフリップが起こっていることが分かります． ここで，ハミング距離 (HD) を考えてみると(差異があるビットの合計数をハミング距離という)，3 だとわかります． この時の消費電力を大雑把に 3P と計算します． また，以下のようにビットフリップが起こったとします． $$(0000 0000) \rightarrow (1111 1110)$$ この HD は 7 なので，大雑把に消費電力は 7P と計算できます． 3P と 7P を比較すると， 7P の方が消費電力が大きいだろうと予測することができます． また，このようにビットフリップと消費電力を関連付けることで，双方から予測することができるようになります． 例えば，まずあらかじめ 1P から 8P までの消費電力を記録しておきます． 次に，ある処理で発生した消費電力を計測し，先ほどの消費電力の記録と照合します． このとき，例えば計測した消費電力が 3P と一致していれば，ビットフリップは 3 bit 分起こったと分かります． ビットフリップ後の値を予測しようと全探索すると 2^8 ですが，消費電力が 3P だとわかっていれば 2^3 で済みます．
以上のことを実際に実験によって確認してみます．
