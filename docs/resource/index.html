電力差分解析 - サイドチャネル攻撃 ⚡

🏝️ ここではサイドチャネル解析 (SCA) について，その概要を紹介します 🏝️ 回路からの情報リーク デバイスの中には回路が実装されています． その回路は論理ゲートから構成されています． 論理ゲートは boolean 変数を扱うので，入力が 1, 0 かつ出力が 1, 0 となります． さらに，論理ゲートは CMOS から構成されており，この CMOS はトランジスタで構成されています． この CMOS はビットフリップが起こると消費電力が増える特性をもちます． 通常動作のとき，つまりビットフリップが起こらないとき (0 -> 0, 1 -> 1) は消費電力が少なく， ビットフリップが起こるとき (0 -> 1, 1 -> 0) に消費電力が増えます． このとき，CMOS 全体の消費電力は，何もしなくても消費する電力 (static 消費電力) と，ビットフリップで消費する電力 (dynamic 消費電力) の和で表されます． $$ P_{CMOS} = P_{static} + P_{dynamic} $$ CMOS の説明 static 消費電力 何もしなくても，電源電圧を印可するだけで CMOS が消費する電力を static 消費電力といいます． ここでキャパシタを考えてみます． キャパシタは，ふたつの離れた導体 (conductor) が違う電位にある場合，両者の間にキャパシタンスが発生します． 導体の間には誘電体 (dielectric) があり，この誘電体には全く電流が流れないことが理想です． しかし，現実の誘電体は完全には絶縁体となっておらず，leakage current と呼ばれる電流がわずかに流れます． MOSFET は，ゲート・ドレイン間およびゲート・ソース間に寄生容量が発生しますが，その間にある絶縁体がわずかに電流を流すことで static 消費電力が発生します． このとき，static 消費電力は以下のように表されます． $$ P_{static} = I * V $$...

電力差分解析 - 共通鍵暗号 👥

🎷 ここでは今回の解析対象である共通鍵暗号について紹介します 🎷 共通鍵暗号の概要 共通鍵暗号を使った通信 アリスとボブがお互いに通信をしたいけれど，二人の間にイヴがいるため，盗聴されたくないという状況を考えます． ここで，通信したいメッセージ，平文を m , アリスとボブが共有している秘密鍵を k とします． このとき，暗号文は，秘密鍵 k を使って以下のような計算で得ることができます． $$ c = E_k(m) $$ この c を受け取ったボブは， k を使って暗号文を以下のように復号します． $$ m = D_k(c) $$ こうしてボブは m を知ることができました． 一方イヴは暗号文 c を見ることができますが，秘密鍵 k を知らないため，復号することができず，m を知ることができません． このように，暗号化に使う鍵と復号に使う鍵が同一である暗号方式を共通鍵暗号といいます． one-time pad one-time pad (Vernam)という共通鍵暗号方式があります． 今回もアリスとボブがお互いに通信したいけれど，二人の間にイヴがいるシチュエーションを考えます． このとき，アリスが伝えたいメッセージ m を暗号化する必要があります． 例えばメッセージと鍵を以下のように仮定します． $$ m = 10101100, k = 01100011 $$ 次にアリスは以下のようにメッセージを暗号化します． $$ c = m~\oplus~k $$ つまり，今回の場合以下のようになります． \begin{eqnarray} c & = & 10101100~\oplus~01100011 \\ &=& 11001111 \end{eqnarray} 暗号文 c を受け取ったボブは，あらかじめ共有されていた k を使って以下のように復号します． $$ m = c~\oplus~k $$ つまり，今回の場合以下のようになります． \begin{eqnarray} m’ & = & 11001111~\oplus~01100011 \\ &=& 10101100 \end{eqnarray} これを見ると，たしかに m = m’ で一致しています． one-time pad は絶対に破られないことが保証されている暗号です． 絶対に破られない，とはどういうことか以下で考えてみます．...

電力差分解析 - 暗号 🔐

🥁 ここでは電力差分解析に必要な暗号について，その概要を紹介します 🥁 暗号について 暗号化 「暗号化」とは，ある二人のやりとりを見ても，当事者以外はその内容が分からないようにする方法のことを言います． 例えば，あなたが授業中，教室で遠く離れた友達に手紙を渡したいとします． このとき友達とあなた以外には手紙の内容を知られたくないのですが，友達に手紙を回すにはたくさんのクラスメイトの手をわたる必要があります． そこで，授業の前の休み時間に，あらかじめ友達とあなただけが知るある「ルール」を共有しておくことにしました． そのルールとは，送りたいメッセージ ( 平文といいます ) で「文字を 50 音の 1 文字前の文字に置き換える」というものです． 例えば「おなかすいた」という平文を，暗号化すると「えとおしあそ」となります ( 暗号化されたあとの文を暗号文といいます ) ． このルールを知っているあなたと友達はその暗号文を平文に戻すことができますが ( 平文に戻すことを復号といいます ) ，このルールを知らないクラスメイトはいくらこの暗号文を見ても平文や暗号方式は分かりません． このように，暗号化をすると，知られたくない人には平文を知られず，知られたい人には平文を知らせることができます． しかしある日，クラスメイトの一人がルールを盗み聞きしてしまいました． これでは平文が知られてしまうので，あなたと友人はルールを変えることにしました． すなわち，「文字を 50 音の x 文字 ( 前 / 後 ) の文字に置き換える」ということだけを公開情報として共有し，これの x と ( 前 or 後 ) だけをあなたと友人で秘密の鍵として共有します． このときルールはクラスメイト全員に知られてもよく，鍵だけはあなたと友人だけが知る秘密の情報とします． 例えば，鍵を ( 3, 後 ) とします． つまり，「さむいね」という平文は「せやおひ」となります． すると，クラスメイトはルールは知っていますが，鍵を知らないので復号することができません． 一方で，あなたと友達はルールも鍵も知っているので復号することができます． 万一鍵を知られてしまったとしても新しく鍵を作って ( 例えば ( 5, 前 ) などに変更する ) しまえばまたクラスメイトは分からなくなります． 鍵は 50 * 2 通りしかないので，全探索することもできますが，全探索の結果の中から意味の通る組み合わせを探すことは大変です． このように，暗号は「ルール」や「暗号文」など鍵以外のパーツを知られても鍵や平文が漏洩しないように設計されています．...
